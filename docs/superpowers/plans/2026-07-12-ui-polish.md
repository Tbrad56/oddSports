## Verification tooling (used by Tasks 1-3)

The audit measured layouts with a probe page served same-origin: an HTML file in `public/` (temporary, deleted before the final commit) that iframes a target page at 390px, waits for load, measures, and writes results into the DOM so `chrome --headless=new --dump-dom` can print them. Recreate as `public/_probe.html`:

```html
<!DOCTYPE html><html><body>
<iframe id="f" style="width:390px;height:844px;border:0"></iframe>
<pre id="out">pending</pre>
<script>
  const page = new URLSearchParams(location.search).get('p') || '/';
  const f = document.getElementById('f');
  f.src = page;
  f.onload = () => setTimeout(() => {
    const d = f.contentDocument;
    document.getElementById('out').textContent = JSON.stringify({
      page, scrollWidth: d.documentElement.scrollWidth,
      bodyScrollWidth: d.body.scrollWidth
    });
  }, 2500);
</script>
</body></html>
```

Run (server on :3000): `chrome --headless=new --dump-dom --virtual-time-budget=8000 "http://localhost:3000/_probe.html?p=/record.html" 2>/dev/null | grep -o '{.*}'` вҖ” the chromium binary the audit used is on PATH as `google-chrome` or `chromium` (check both). Loaded-state pages need data: for Record, seed via `DATA_DIR` fixtures or accept the empty state plus a DOM-injected fixture (the audit's stub approach); document in the report which pages were probed loaded vs empty. `public/_probe.html` must NOT be in the final commit of any task (`git status` check).
